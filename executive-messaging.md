### **The Architecture We Use:**

Recent AI research (CHC intelligence studies) shows: LLMs excel at reasoning and language but struggle with working memory and persistent context.

The solution isn't "better prompts." It's giving AI external memory through IDE file systems.

But the models needed to be good enough first. Mid-2024: just code completion. Mid-2025 (Claude Sonnet 4, GPT-4.x): better but frustrating - brilliant code that misunderstood intent, made string escaping errors, broke things during refactoring.

October 2025 changed everything. Claude Sonnet 4.5 and GPT-5 finally understand intent reliably. They don't make junior mistakes. You can trust them with refactoring. Combined with VS Code improvements, they're ready for business experts.

**How it works:**
- Your requirements → persistent documentation files (AI references these across sessions)
- Your business rules → configuration files (context that never gets lost)
- Your edge cases → test files and comments (learning that compounds)
- Each refinement adds context, never loses it

**The model strategy:**
- Use GPT-5 for research phase (brilliant at synthesizing information, asking clarifying questions)
- Use Claude Sonnet 4.5 for building phase (exceptional at code structure, handles complexity better)
- IDEs like VS Code + Copilot maintain context between both

Why this matters: ChatGPT degrades across iterations. IDE-based AI with Claude 4.5/GPT-5 compounds improvements. This unlocks business-scale complexity for non-developers.

We teach this cognitive architecture and model selection strategy, not just "how to use AI."
